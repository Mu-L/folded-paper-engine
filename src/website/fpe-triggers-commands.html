Triggers and Commands
Area sensors, trigger filters, and built-in actions

Tracking areas

  • Any object with trigger data spawns a TrackingArea3D that copies the mesh bounds (or a simple box) and watches for RigidBody3D bodies.
  • Player bodies record entered areas so prompts or inventory pickups can react to ENTER and EXIT trigger types.
  • Prefer simple box shapes for performance when a detailed mesh collider is unnecessary.

Trigger filters

  • Use trigger groups to limit which initiators can fire an event. The engine checks group compatibility before dispatching.
  • Inventory-bearing triggers can hold InventoryItemSlot data so stepping on the trigger transfers quantities into the player inventory.

Command catalog

  • LoadLevel: unload current GLB and load the target path.
  • Animations / StopAnimations: play or stop comma-separated animation names on registered animation players and sprite animators.
  • SpeakerTrigger / SpeakerTriggerSelf: play a named speaker or the speaker attached to the trigger node.
  • ActivateCamera / ReactivatePlayerCamera: switch to a named camera or return control to the player camera.
  • Deactivate/ReactivatePlayerControls: globally gate player input when cutscenes or menus run.
  • LoadSubScene / UnloadSubScene / UnloadThisSubScene: stream sub-scenes in or out by name.
  • StartConversation: start a conversation using the provided array of conversation data.
  • Pause/ResumeSpecificActivities: toggle activity types using numeric IDs.
  • DispatchEvent: emit a named FPEEvent with participant metadata gathered from owner/initiator user data.
  • DeleteByGroup: queue free all nodes in a given group.
