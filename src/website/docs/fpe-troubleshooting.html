Troubleshooting
Common pipeline issues and fixes

GLB import issues

  • Missing behaviors: ensure Custom Properties are enabled in the Blender export dialog so metadata reaches Godot.
  • Stale data: after re-exporting, let Godot reimport the GLB (or delete the .import folder for the file) to refresh metadata.
  • Wrong scale/origin: apply transforms in Blender before export; FPE uses the mesh to build triggers and colliders.

Runtime hiccups

  • No input: verify the plug-in is enabled and that the engine node’s capture_pointer matches your test setup. Controller hot-plugging relies on the input manager being active.
  • Music or speakers silent: confirm paths are correct and that autoplay is set if you expect immediate playback. Speakers clear when nodes are removed, so check node names match command targets.
  • Triggers not firing: make sure trigger groups match and the initiator is a RigidBody3D. Use simple box triggers for empty meshes.

Level flow

  • Scene reload glitches: register unload procedures for custom systems, and avoid storing references to nodes that will be freed during global_unload_level.
  • Paused controls: if movement is frozen, check the global deactivate flags (player controls, character movement, triggers, UI). Commands may have toggled these during cutscenes.
