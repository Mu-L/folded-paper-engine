UI and Cursors
Build menus and prompts with UI Elements

Mark UI Elements in Blender

  • Enable UI Element in the Folded Paper Engine panel on the object you want as HUD or menu content. This flag simply makes the imported node eligible to be a UI Cursor or a UI Option.
  • Keep UI meshes inside the camera’s view and parent them appropriately (for example, under a canvas or a camera) so they stay visible; the UI Element tag does not automatically reposition them.
  • Apply scale and transforms before export so the cursor or option collider matches the visuals in Godot.

Set up a UI Cursor

  • Turn on UI Cursor for a UI Element to spawn a 3D cursor node in Godot. You can adjust cursor depth, the select animation, and whether the cursor faces the camera.
  • Each cursor receives its own input device proxy, so mouse, keyboard, or gamepad users can all navigate UI without extra wiring.
  • Lock or unlock the mouse using your pause/menu flow as needed—the cursor will respect your camera and still track UI Options that remain in view.

Make interactive UI Options

  • Enable UI Option on a UI Element to turn it into a selectable hotspot. The engine adds it to the global UI option list so any UI Cursor or scripted raycast can highlight and trigger it.
  • Attach commands to the option (for example, Start Conversation, Load Sub Scene, or a custom trigger) so clicking or selecting the option runs gameplay actions.
  • Use the UI Option Command Delay setting if you want a short pause after a selection to play an animation before the command fires.
