UI, Cursors, and On-Screen Options
Creator-friendly crosshairs, menu options, and prompts

Mark a UI Element in Blender

  1. Select the mesh you want to use as a cursor or clickable option.
  2. Open the UI Element panel and choose either UI Cursor or UI Option. These are the only behaviors UI Elements provide.
  3. Keep the object in view with your camera framing—UI Elements do not auto-anchor to the screen, so plan your camera rigs and animations so the UI stays visible.

Create a cursor

  • Enable UI Cursor to let the engine track and display the object as an interactive cursor.
  • Set UI Cursor Depth if you want the cursor to float in front of the camera, and enable UI Cursor Look At Camera if it should always face the player.
  • Optionally provide a UI Cursor Select Animation name and play that animation when the player selects a UI Option.

Create UI options

  • Enable UI Option on the meshes you want to select with the cursor. These options are stored globally so menus can read them at runtime.
  • Use UI Option Command Delay to postpone a command after selection—helpful for playing a click animation before the action runs.
  • Pair options with triggers or commands (like Load Level or Start Conversation) so selections do something in the world.

Input, prompts, and accessibility

  • Show an on-screen prompt (“Press Use”) when the player is inside a trigger that will open a menu or run a UI Option. This keeps non-technical players oriented.
  • If you temporarily unlock the mouse for menus, remember to restore pointer capture when closing the UI so gameplay controls resume.
  • Group your UI Option nodes for easy enabling/disabling in Godot when switching between menus.
