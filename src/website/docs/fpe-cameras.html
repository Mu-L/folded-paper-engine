Cameras
Registering, switching, and returning to the player view

How cameras are registered

  • Every Camera3D in your imported GLB is cataloged by name when the scene is processed, so you can target it later without writing code. Keep names readable in Blender or Godot—the name becomes the key in the scene camera map. 【F:src/Game/addons/folded_paper_engine/Engine/GLTFModelUtils.gd†L59-L78】
  • Player cameras are also tracked globally so the engine remembers which gameplay view to return to after cutscenes or scripted switches. 【F:src/Game/addons/folded_paper_engine/Engine/FPEGlobals.gd†L4-L17】

Activating a specific camera

  • Use the Activate Camera command to make any named camera current—for example, pair it with a Trigger to jump to a cinematic angle. 【F:src/Game/addons/folded_paper_engine/Engine/Features/Commands/ComandContants.gd†L3-L20】【F:src/Game/addons/folded_paper_engine/Engine/Cameras/CameraUtils.gd†L3-L8】
  • When a command runs, the camera utility looks up the name you provide in the scene camera list and calls make_current() so Godot switches views immediately. 【F:src/Game/addons/folded_paper_engine/Engine/Cameras/CameraUtils.gd†L3-L8】

Returning control to the player

  • Trigger the Reactivate Player Camera command to hand control back to the most recent player camera. This is handy after cutscenes or sub-scenes. 【F:src/Game/addons/folded_paper_engine/Engine/Features/Commands/ComandContants.gd†L3-L20】【F:src/Game/addons/folded_paper_engine/Engine/Cameras/CameraUtils.gd†L10-L17】
  • The utility grabs the latest player camera from the global list and calls make_current(), so you do not need extra logic if you never deactivated the player view. 【F:src/Game/addons/folded_paper_engine/Engine/Cameras/CameraUtils.gd†L10-L17】
