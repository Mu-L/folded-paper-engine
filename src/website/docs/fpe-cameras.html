Cameras
Player rigs, cinematic cuts, and keeping UI in view

Player cameras

  • Players set to First Person use the camera aligned to their head/eyes and lock pitch to keep motion natural.
  • Players set to Third Person orbit around the character with an over-the-shoulder offset. You can tweak offsets in Godot after import.
  • The runtime registers the active player camera globally so you can switch away and come back with commands.

Switching cameras in Godot

  1. Add extra cameras for cutscenes or special views and give them clear names.
  2. Use Activate Camera to cut to a named camera, then Reactivate Player Camera to return. If you never deactivated the player camera, you do not need to reactivate it.
  3. Pair camera switches with Deactivate Player Controls if you want the player to stop moving during the shot.

UI visibility while switching

  • Keep any UI Element children (crosshairs, prompts) parented to the active camera. When you switch cameras, move or duplicate the UI Element so it stays in view.
  • If you cut to a camera that looks at the player, remember to hide first-person-only UI like a crosshair to avoid floating UI in the shot.

Checks if cameras misbehave

  • Ensure the Folded Paper Engine node is in the scene; it registers cameras and handles pointer capture.
  • If the wrong camera stays active after a cutscene, confirm your commands call Reactivate Player Camera only when you actually switched away.
