Installation and Project Setup
Blender exporter + Godot runtime

Follow these steps in order: install the Blender add-on, install the Godot plug-in, then wire the two together with an exported GLB. Everything below matches the labels you will see in Blender and Godot.

Blender add-on (step-by-step)

  1. Download the Blender add-on ZIP from the Folded Paper Engine website.
  2. Open Edit → Preferences → Add-ons, click the arrow in the upper right, choose Install from Disk…, and select the ZIP.
  3. Enable the add-on. Press N in the 3D View and look for the Folded Paper Engine panel to confirm it loaded.
  4. When exporting (File → Export → glTF 2.0), open Include and check Custom Properties as well as Cameras and Punctual Lights. Turn on Remember Export Settings so future exports stay correct.

Godot plug-in (step-by-step)

  1. Download the Godot add-on from the website.
  2. In Godot, open the Asset Library, choose Import, select the add-on, and click Install. Then open Project → Project Settings → Plugins and enable Folded Paper Engine. Restart Godot.
  3. Create a new 3D scene. Add a Folded Paper Engine node, a Sun, and a WorldEnvironment.
  4. In the Folded Paper Engine inspector, set Path to your exported .glb/.gltf file and set Environment to your WorldEnvironment node. Run the scene to see player controls, cameras, and behaviors come online.

Project hygiene

  • Keep exported GLB/GLTF files in your Godot project folder and under version control so the importer stays in sync.
  • After re-exporting from Blender, let Godot reimport the file (or clear the .import folder for that GLB) before testing.
  • Double-check the plug-in is enabled when switching machines or pulling fresh clones—behavior registration depends on it.
