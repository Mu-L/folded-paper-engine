Installation and Project Setup
Blender exporter + Godot runtime

Blender add-on

  1. Open Edit → Preferences → Add-ons in Blender.
  2. Click the upper-right arrow and choose Install from Disk…, then select the Folded Paper Engine add-on ZIP (download it from the site root links).
  3. Enable the add-on. Press N to confirm the Folded Paper Engine panel appears.
  4. Enable Custom Properties, Cameras, and Punctual Lights when exporting to GLB/GLTF so FPE metadata travels with the file.
  5. Video reference: Blender install short (matches these steps).
Version compatibility
  • Use the add-on build that matches your Godot plug-in release to keep feature flags in sync.
  • After Blender updates, re-enable the add-on if preferences reset.

Godot plug-in

  1. Download the Godot add-on from the website and open Godot.
  2. Go to AssetLib → Import, pick the add-on ZIP, click Install, then enable it under Project → Project Settings → Plugins.
  3. Create a new 3D scene, add a Folded Paper Engine node plus a WorldEnvironment, and use the document icon to point Path at your main .glb/.gltf.
  4. Assign your WorldEnvironment resource to the Environment slot. Add a Sun light for quick testing.
  5. Video reference: Godot install short (walks through the same flow).
Importer refresh checklist
  • When a GLB changes, let Godot finish reimporting (progress bar in the bottom panel) before running the scene.
  • If metadata looks stale, right-click the GLB and choose Reimport, or delete its .import folder and reopen the project.
  • After adding the plug-in to an existing project, restart the editor once so the engine node registers signals and inputs.

Project hygiene

  • Keep exported GLB/GLTF files under version control so Godot’s importer can rebuild instances consistently.
  • When updating a GLB, re-export from Blender with transforms applied and let Godot reimport before running.
  • Verify the plug-in is active any time you clone the repo or switch machines; FPE nodes rely on the runtime scripts being enabled.
  • Set a .gitattributes rule for binary GLB files if you use Git LFS so large exports don’t break your history.
