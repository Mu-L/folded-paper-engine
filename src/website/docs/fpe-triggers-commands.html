Triggers and Commands
Step-by-step areas, filters, and the no-code action list

Create a trigger zone in Blender

  1. Select or add a simple cube/empty for your trigger. Scale it to cover the area you want to react to (e.g., a doorway).
  2. Open the FPE panel and check Trigger.
  3. Choose the trigger Type (ENTER, EXIT, or both) so you know when it fires.
  4. Optional: set a Group name if only certain objects should activate it (like Player or Enemy).
  5. Export to GLB. FPE will spawn an invisible TrackingArea3D with a simple box collider for performance.

Bind a trigger to an action without code

  1. In the same Trigger block, add a Command row. You can stack multiple commands in order.
  2. Pick from the built-in list below. Fill in the friendly fields (target names, file paths, etc.).
  3. Re-export and run the scene. Walk into the area to test the action.

Popular commands you can chain

  • LoadLevel: Switch to another GLB/scene. Perfect for doors and portals.
  • LoadSubScene / UnloadSubScene / UnloadThisSubScene: Stream a room or interior in/out while staying in the same level.
  • SpeakerTrigger / SpeakerTriggerSelf: Play a specific Speaker by name, or the Speaker on the trigger node.
  • Animations / StopAnimations: Play or stop comma-separated animation names (works with frame events too).
  • ActivateCamera / ReactivatePlayerCamera: Cut to a cinematic camera, then give control back.
  • Deactivate/ReactivatePlayerControls: Temporarily pause movement during menus or cutscenes.
  • StartConversation: Begin a dialogue using the conversation data on your Speaker/NPC.
  • Pause/ResumeSpecificActivities: Freeze physics or movement types by ID while something else runs.
  • DispatchEvent: Emit a custom event name that other triggers or scripts can listen to.
  • DeleteByGroup: Remove every node in a group (useful for clearing props after a puzzle).

Inventory-aware triggers

  • Add an Inventory Item Slot to the trigger to let the player pick up items just by stepping into the zone.
  • Use the Group filter to make sure only the player activates item pickups, not a rolling crate.

Design patterns to copy

  • Pressure plate puzzle: Trigger on ENTER → play a click sound (SpeakerTrigger), open the door (Animations), and disable itself (DeleteByGroup on a helper group).
  • Cutscene starter: Trigger on ENTER → DeactivatePlayerControls → ActivateCamera → run an Animations clip with frame events → ReactivatePlayerCamera → ReactivatePlayerControls.
  • Scene exit: Trigger on ENTER → LoadLevel with the next GLB path. Pair with a fade or loading screen sub-scene for polish.

Troubleshooting triggers

  • If nothing fires, confirm the trigger cube actually covers the player capsule and that transforms are applied before export.
  • Check the Group filter: wrong group means the engine ignores the overlap.
  • Use simple box triggers unless you truly need mesh-shaped bounds for performance reasons.
