Input and Cameras
Action maps, controllers, and camera switching

Input actions

  • Player actions include movement (PLAYER_MOVE_*), look (PLAYER_TURN_*), run, jump, use, and drop. UI actions cover navigation, accept/cancel, and pause.
  • Device proxies normalize mouse, keyboard, and gamepad inputs. Mouse axes respect a configurable ratio to balance with joystick sensitivity.
  • Input mappings update when controllers connect/disconnect so players can hot-plug devices mid-session.

Camera control

  • Player cameras are registered globally. Commands can activate any named camera or reactivate the player camera after cutscenes.
  • Third-person cameras align to movement and can be offset for over-the-shoulder views; first-person locks facing and clamps pitch to ±90°.
  • UI or cutscene scripts can temporarily disable player controls via activity control utilities, preventing unintended movement while cameras animate.
