Speakers, looping, autoplay, and background music

Speakers on objects (Blender setup)

  1. Select the object that should play a sound (an ambient prop, a humming machine, an NPC, etc.).
  2. In the Folded Paper Engine panel, check the Speaker box (this creates a 3D audio player when the scene loads in Godot).
  3. Set SpeakerFile to the audio file name you will place in your Godot project (for example, sounds/vent_hum.ogg).
  4. Optional: turn on SpeakerAutoplay to start playing as soon as the level loads, and SpeakerLoop if it should repeat. You can also adjust SpeakerVolume (dB) and SpeakerMaxDistance for how far the sound carries.
  5. Export the GLB and import it into Godot. The FPE plugin spawns an AudioStreamPlayer3D on that object, wires up the loop/autoplay settings, and removes it cleanly when the node is freed.

Background music (scene-wide)

  1. Select the Scene settings object in Blender.
  2. Add BackgroundMusic with the track name you will place in Godot (for example, music/forest_theme.ogg).
  3. Use BackgroundMusicVolume to set the starting loudness. FPE queues and starts the track after the level loads, and stops it automatically when you change scenes.

Triggering audio with interactions

  1. Create a trigger volume (or use an animation frame event) and call SpeakerTrigger or SpeakerTriggerSelf to start or stop a sound on demand.
  2. Common pattern: pair a trigger with a Speaker object to start a loop when the player enters an area, then stop it when they leave.
