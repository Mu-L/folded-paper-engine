Inventory and Holdables
Two separate systems: backpack slots vs. items you can carry by hand

Inventory items (backpack slots)

  1. In Blender, select your item and check Trigger under Item so it can be detected in-game.
  2. Open the Folded Paper Engine panel → Inventory Item and set the friendly Inventory Item Kind plus the Quantity to award.
  3. Duplicate the object if you want several pickups; each copy keeps the same kind and quantity.
  4. In Godot, add an FPE Inventory Config node. In the inspector, add an Inventory Item Kind entry and make sure its ID matches the kind you set in Blender. Set an Inventory Size so the backpack has enough slots.

Inventory items live in slots and can be given to the player automatically when they enter a trigger, or traded with deposits.

Holdable items (hands-on carrying)

  1. Select your player in Blender → Folded Paper Engine panel → Player Controls. Choose First Person (or your preferred mode) and enable Can Hold Items.
  2. Make the item you want to hold a rigid body so it has proper physics.
  3. Select the item and, under the Folded Paper Engine tab, check Holdable so it can be picked up into the Hold Zone.
  4. In-game, the player presses the Use action to grab or drop the item. The Hold Zone keeps it in front of the camera with gentle sway and will drop it if it drifts too far.

Holdables are physical objects you carry in front of you; they are separate from inventory slots and do not require an Inventory Config.

Deposits and pickups

  • Triggers can grant inventory items directly to the player. Overflow stays on the trigger until there is room.
  • Place a deposit volume where you want to trade items; Hold Zones watch nearby deposits so players can drop or withdraw items while standing in the area.
