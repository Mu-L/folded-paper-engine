Inventory and Holdables
Item kinds and quantities vs. physics-held objects

Inventory items (stored in slots)

  1. Model your item in Blender, then in the Folded Paper Engine panel open Inventory Item and check Inventory Item.
  2. Set the Inventory Item Kind (a friendly ID like Key or Coin) and the Quantity that should be added when the item is picked up.
  3. If you want the pickup to run when the player enters a zone, check Trigger under Item so the item acts as a trigger volume.
  4. Duplicate the item as needed. Each copy will add the same kind/quantity to the player when triggered.
  5. In Godot, add an FPE Inventory Config node to your scene, then add the same Inventory Item Kind IDs you used in Blender and set your Inventory Size. This tells the runtime which kinds exist and how many slots the player has.
  6. Playtest: walking into the item trigger moves the configured quantity into the inventory slots. The stored items can be shown in UI or spent later via triggers/commands.

Holdable items (physically carried)

  1. Select your player in Blender, open Player Controls, choose First Person, and check Can Hold Items so the exporter knows to create a Hold Zone.
  2. Pick the object you want to carry, make sure it is marked Rigid so physics work, then open the Items tab and check Holdable.
  3. Export to GLB. In-game, pressing the Use action pulls the holdable object into the Hold Zone in front of the camera instead of putting it into an inventory slot.
  4. Holdables keep their physics: they sway, can be dropped with gravity, and do not consume inventory capacity. Use them for boxes, tools, or props you want the player to visibly carry.

Pickup and UI patterns

  • Add a UI Options node and enable Show Pickup Prompts if you want on-screen text when the player can take an item.
  • For slot items, consider a brief animation or sound on pickup so it is clear the item was stored, not held.
  • If you allow dropping, keep a Holdable version of the model nearby so players can set objects down even when inventory is full.

Choosing between them

  • Inventory Items live in slots with a Kind and Quantity. Use them for resources, keys, and collectibles that you do not need to see in the player’s hands.
  • Holdable Items stay in the world as physics bodies and can be picked up/dropped with the player’s hands. They are great for puzzles and props but they do not consume or require inventory slots.
  • You can mix both: keep a key Inventory Item for a door while letting the player carry a separate Holdable crate to press a switch.
