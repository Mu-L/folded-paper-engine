Inventory and Holdables
Two different systems: stacked items in slots vs. physics-based carry

How they differ

Inventory tracks kinds and quantities in slots (great for keys, ammo, or resources).
Holdable Items are physical objects the player lifts with a hold zone (great for boxes or props).
You can use both: pick up a Holdable, drop it onto a Trigger to deposit it, or keep inventory for stacking.

Inventory setup (Blender)

  1. Create your item mesh. In the Item panel, check Trigger so the player can interact.
  2. Still in the Item panel, open Inventory Item and fill:
    • Inventory Item Kind: a text ID that represents this item type (for example, Key_A).
    • Inventory Item Quantity: how many units this trigger should give (for example, 1 key or 5 coins).
  3. Duplicate the object if you want multiple pickups. Export as usual.

Inventory setup (Godot)

  1. In your imported scene, add a child node FPEInventoryConfig (appears after enabling the plugin).
  2. In the Inspector:
    • Inventory Kinds: click + to add an item kind entry, set its ID to exactly match the Inventory Item Kind you set in Blender.
    • Player Inventory Size: set Width (columns) and Height (rows) to control slot count.
    • Keep Player Inventory: enable if you want the player to keep items when a level unloads.
  3. Run the scene. Walking into an Inventory trigger adds the quantity to the player’s slots. Overflow stays on the trigger.

Holdable Items (Blender)

  1. Select your player object. In Player Controls, enable First Person and Can Hold Items. Optional: adjust Hold Zone Distance, Hold Zone Size, or pick a Hold Zone Scene for VFX/UI feedback.
  2. Select the object you want to carry:
    • Give it rigid-body physics (so it can be pushed and lifted).
    • In the Item panel, check Holdable.
  3. Export and import into Godot.

Using Holdables in-game

  • With First Person and Can Hold Items enabled, the player’s Use input grabs nearby Holdable objects into a physics-based hold zone in front of the camera.
  • The hold zone gently pulls items toward the target point, applies sway, and drops them if they get too far or hit the ground.
  • Drop a held item over a Trigger volume to deposit it (for example, into a puzzle socket) or just let it fall to place it in the world.

Combine both systems (optional)

Use inventory for stackable resources and Holdables for physical props. A level can include both: collect coins and keys into the inventory while also lifting crates or quest items with the hold zone. You decide which objects should be stackable (Inventory Item) versus physics carry (Holdable) for your design.
