Animation and Frame Events
Animation players, sprite animations, and timed events

Animation import

  • Animations are registered per name when the GLB loads; players are stored in a global map for command access.
  • Sprite animations are also indexed so command-driven playback can start or stop either type.

Frame events

  • A FrameEventRunner scans animation metadata for frame markers. Events include a frame number/time and a named event string.
  • While an animation plays, the runner checks for pending frame events within a small tolerance and dispatches FPEEvent instances to the target node path.
  • Pair frame events with trigger or command handlers to synchronize sounds, gameplay state changes, or camera cuts with keyframes.

Playback control

  • Use commands to play or stop animations by comma-separated names. Both mesh and sprite animations listen to the same command list.
  • Animation players are reused across levels; unload clears the map so stale references do not persist.
