Sub Scenes and Level Loading
Friendly recipes for doors, hubs, and streamed interiors

One-click whole level transitions

  1. Place a trigger near your doorway or portal (see Triggers guide).
  2. Add a Load Level command and point it at the next GLB path (relative to your project).
  3. When fired, Folded Paper Engine unloads the current level, clears global registries (speakers, cameras, animations), and loads the new GLB cleanly.
  4. Tip: pair with a quick fade or loading screen sub-scene so the player sees a smooth handoff.

Quick fade recipe

  • Create a simple sub-scene with a full-screen quad and an AnimationPlayer that animates opacity over 0.3–0.5s.
  • Load that sub-scene before calling Load Level, play the fade-in/out animation, and unload it after the new level is ready.
  • If you prefer commands, queue Load Sub Scene for the fade, then run Load Level, and finally Unload Sub Scene once the fade completes.

Adding a Sub Scene host in Blender

  1. Select the mesh that should act as the placeholder (e.g., a building doorway or elevator cage).
  2. Open the Folded Paper Engine panel and check Sub Scene.
  3. Provide the File Path to the GLB you want to stream in. You can also name the sub-scene so commands can target it later.
  4. Decide if the host should Auto Load on start. If unchecked, load it later via a trigger or an Event Command.
  5. Export to GLB. In Godot, the host replaces its mesh with the streamed content while keeping its transform.

Commands for streaming

  • Load Sub Scene: Load a sub-scene by name or host node path. Great for stepping inside a house while keeping the street loaded.
  • Unload Sub Scene: Unload a named sub-scene and optionally wait a few seconds to finish fades.
  • Unload This Sub Scene: From inside a sub-scene, call this to close yourself (handy on an exit trigger inside the streamed area).

Pause control during transitions

  • Sub scene hosts can pause parent activities while the child is loaded. That keeps NPCs or physics from running while the player is away.
  • Use Deactivate Player Controls plus a camera cut to hide loading, then Reactivate Player Controls when ready. You only need to reactivate a camera if you deactivated it earlier.

Design templates

  • Interior door: Trigger on the doorframe → Load Sub Scene for the interior → on the interior exit trigger, run Unload This Sub Scene (and reactivate the player camera only if you turned it off).
  • Hub world with portals: Each portal trigger runs Load Level to another GLB. Keep a shared UI scene loaded to show tips during transitions.
  • Elevator cutscene: Trigger → Deactivate Player Controls → play lift animation with frame events → Load Level to the next floor.

Checks if something feels broken

  • Verify paths: GLB paths are relative to the project. Typos silently skip loading.
  • Apply transforms before export so the sub-scene spawns at the expected size and position.
  • If old sounds or cameras keep playing after a load, make sure you are using Load Level (not a custom script) so the runtime clears registries.
