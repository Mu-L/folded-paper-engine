Sub Scenes and Level Loading
Friendly recipes for doors, hubs, and streamed interiors

One-click whole level transitions

  1. Place a trigger near your doorway or portal (see Triggers guide).
  2. Add a Load Level command (the LoadLevel event) and point it at the next GLB path (relative to your project).
  3. When fired, FPE unloads the current level, clears global registries (speakers, cameras, animations), and loads the new GLB cleanly.
  4. Tip: pair with a quick fade or loading screen sub-scene so the player sees a smooth handoff.

Quick fade recipe

  1. Create a simple full-screen quad or UI Layer that fades from transparent to black over 0.25–0.5 seconds.
  2. Trigger the fade at the same time you run LoadLevel or LoadSubScene. Start the fade-in once the new content is ready.
  3. You can keep the fade as a lightweight sub-scene that is always loaded, or load/unload it around transitions for a blank-free swap.

Adding a Sub Scene host in Blender

  1. Select the mesh that should act as the placeholder (e.g., a building doorway or elevator cage).
  2. Open the Folded Paper Engine panel and check Sub Scene.
  3. Provide the File Path to the GLB you want to stream in. You can also name the sub-scene so commands can target it later.
  4. Decide if the host should Auto Load on start. If unchecked, call Load Sub Scene (the LoadSubScene event) from a trigger or other event command when you want it to appear.
  5. Export to GLB. In Godot, the host replaces its mesh with the streamed content while keeping its transform.

Commands for streaming

  • Load Sub Scene (LoadSubScene): Load a sub-scene by name or host node path. Great for stepping inside a house while keeping the street loaded.
  • Unload Sub Scene (UnloadSubScene): Unload a named sub-scene and optionally wait a few seconds to finish fades.
  • Unload This Sub Scene (UnloadThisSubScene): From inside a sub-scene, call this to close yourself (handy on an exit trigger inside the streamed area).

Quick fades for smooth transitions

  1. Create a simple fade overlay in Blender: a plane in front of the player camera with a dark material set to transparent. Keyframe its material alpha from 0 → 1 (fade out) and 1 → 0 (fade in) so the animation is baked into the GLB.
  2. Parent that plane to the camera you will use during the transition. This keeps the overlay in view without special UI options.
  3. Export the fade overlay as a tiny sub-scene GLB and give it a clear name (for example, fade_overlay).
  4. Before you run Load Level or Load Sub Scene, trigger the overlay’s fade-out animation with the Animations command. When the screen is fully dark, run the load command, then play the fade-in animation. Frame events are a clean way to sequence “fade complete → load → fade back.”

Pause control during transitions

  • Sub scene hosts can pause parent activities while the child is loaded. That keeps NPCs or physics from running while the player is away.
  • Use Deactivate Player Controls (the DeactivatePlayerControls event) plus a camera cut to hide loading, then Reactivate Player Controls when ready.

Design templates

  • Interior door: Trigger on the doorframe → run Load Sub Scene (LoadSubScene) for the interior → on the interior exit trigger, run Unload This Sub Scene (UnloadThisSubScene) and, if you swapped cameras, Reactivate Player Camera (ReactivatePlayerCamera).
  • Hub world with portals: Each portal trigger runs Load Level (LoadLevel) to another GLB. Keep a shared UI scene loaded to show tips during transitions.
  • Elevator cutscene: Trigger → Deactivate Player Controls (DeactivatePlayerControls) → play lift animation with frame events → Load Level (LoadLevel) to the next floor.

Checks if something feels broken

  • Verify paths: GLB paths are relative to the project. Typos silently skip loading.
  • Apply transforms before export so the sub-scene spawns at the expected size and position.
  • If old sounds or cameras keep playing after a load, make sure you are using LoadLevel (not a custom script) so the runtime clears registries.
  • After a LoadLevel, clear any trigger states that were mid-activation so you do not immediately re-trigger when the player respawns.
  • When streaming interiors, keep the exterior WorldEnvironment loaded or swap to a dedicated indoor profile so lighting transitions feel intentional.
