Sub Scenes and Level Loading
Friendly recipes for doors, hubs, and streamed interiors
Scene loading essentials
  • Keep a root Folded Paper Engine node active and load sub-scenes under it so feature registration stays stable.
  • Pair level changes with commands that fade audio, detach holdables, and clean up triggers before unloading.
  • Use a lightweight loading or fade sub-scene to mask transitions so players never see streaming artifacts.

One-click whole level transitions

  1. Place a trigger near your doorway or portal (see Triggers guide).
  2. Add a LoadLevel command and point it at the next GLB path (relative to your project).
  3. When fired, FPE unloads the current level, clears global registries (speakers, cameras, animations), and loads the new GLB cleanly.
  4. Tip: pair with a quick fade or loading screen sub-scene so the player sees a smooth handoff.

Quick fade recipe

  • Create a simple full-screen plane or color-rect UI Element in Blender (black or dark gray works well) and animate its opacity.
  • Export it as a sub-scene and trigger LoadSubScene to show the fade, then run LoadLevel. When the new level is ready, animate the fade back down and unload the sub-scene.
  • This keeps the player from seeing mid-load pops while staying fully no-code for Creator Users.

Adding a Sub Scene host in Blender

  1. Select the mesh that should act as the placeholder (e.g., a building doorway or elevator cage).
  2. Open the FPE panel and check Sub Scene.
  3. Provide the File Path to the GLB you want to stream in. You can also name the sub-scene so commands can target it later.
  4. Decide if the host should Auto Load on start. If unchecked, load it later with a Trigger or an Event Command.
  5. Export to GLB. In Godot, the host replaces its mesh with the streamed content while keeping its transform.

Commands for streaming

  • LoadSubScene: Load a sub-scene by name or host node path. Great for stepping inside a house while keeping the street loaded.
  • UnloadSubScene: Unload a named sub-scene and optionally wait a few seconds to finish fades.
  • UnloadThisSubScene: From inside a sub-scene, call this to close yourself (handy on an exit trigger inside the streamed area).
Cleanup tips
  • Disable or delete triggers tied to an unloaded area so they do not fire twice if the player re-enters quickly.
  • Move reusable fades or HUD overlays into a shared sub-scene instead of duplicating them in every level.
  • For large hubs, unload distant interiors as soon as the exit trigger fires to keep memory predictable.

Pause control during transitions

  • Sub scene hosts can pause parent activities while the child is loaded. That keeps NPCs or physics from running while the player is away.
  • Use DeactivatePlayerControls plus a camera cut to hide loading, then ReactivatePlayerControls when ready.

Design templates

  • Interior door: Trigger on the doorframe → LoadSubScene for the interior → on the interior exit trigger, run UnloadThisSubScene and, if you disabled it, ReactivatePlayerCamera.
  • Hub world with portals: Each portal trigger runs LoadLevel to another GLB. Keep a shared UI scene loaded to show tips during transitions.
  • Elevator cutscene: Trigger → DeactivatePlayerControls → play lift animation with frame events → LoadLevel to the next floor.

Checks if something feels broken

  • Verify paths: GLB paths are relative to the project. Typos silently skip loading.
  • Apply transforms before export so the sub-scene spawns at the expected size and position.
  • If old sounds or cameras keep playing after a load, make sure you are using LoadLevel (not a custom script) so the runtime clears registries.
