Input and Controls
Action maps, devices, and accessibility for players

Turn on player controls in Blender

  1. Select your player rig and open the Player Controls panel.
  2. Choose First Person or Third Person (or both if you plan to switch at runtime).
  3. If you want pickups, enable Can Hold Items and adjust Hold Zone Distance and Hold Zone Size so reach feels natural.

Default actions to know

  • Movement: Player Move (WASD/left stick) and Player Turn (mouse/right stick) are mapped out of the box.
  • Use / Interact: bound to the Use action for triggers, pickups, and UI selections.
  • Jump / Run: available when Character Controls are active on the player.
  • Drop: lets the player release a holdable item if Can Hold Items is checked.

Controller and mouse handling

  • Device proxies normalize keyboard, mouse, and gamepad so you can hot-plug controllers while testing.
  • Mouse look uses a configurable ratio to balance sensitivity with joystick turning.
  • If you unlock the cursor for menus, recapture it when closing the UI so movement resumes.

Adjust input mappings in Godot

  • Open Project → Project Settings → Input Map to remap actions like Use, Jump, or Drop for your players.
  • Add gamepad-friendly bindings for living room setups, then test by hot-plugging a controller while the scene is running.
  • Keep prompts consistent with your mappings (for example, update on-screen text if you swap Use from E to another key).
