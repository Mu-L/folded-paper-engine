Input
Action maps, controller support, and player-friendly prompts

Default actions

  • FPE ships with sensible defaults for Move, Look, Jump, Use, Inventory, and Toggle Menu. Map them in Godot’s Input Map before running.
  • Use clear on-screen prompts (e.g., “Press Use”) whenever a Trigger or UI Option requires a specific action.

Controllers and rebindings

  1. Godot automatically picks up gamepads that map to the standard actions. Test navigation, jumping, and using items with a controller.
  2. If you add new actions, keep the names readable (e.g., Flashlight instead of a code-heavy label) so Creator Users can track them in the UI.
  3. Offer a quick settings menu for sensitivity and inversion; these values can live alongside other UI Options.

Pausing and menus

  • Bind a Pause or Options action to unlock the cursor and show your menus. Returning to the game locks the pointer and re-enables look input.
  • Keep menu navigation simple: Up/Down/Confirm/Back should mirror keyboard, mouse, and controller inputs.
