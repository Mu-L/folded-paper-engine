Conversations
Attach dialogue files to characters and start them from triggers

Build conversation files in Godot

  • Create a new Conversation resource in Godot for each dialogue. Give it a unique ID and pick an initial Comment; each comment contains reply options and optional event data.
  • Add Reply entries under each comment to form branches. Replies point to the next comment, letting you build linear or branching conversations without scripting.
  • If a comment should fire game logic (opening a door, giving an item), set the comment’s event type and data so the engine can dispatch the linked command when that line shows.

Place the Conversation Manager

  • Add an FPE Conversation Manager node to your main scene. It tracks the active conversation, raises signals when the line changes, and makes sure only one conversation runs at a time.
  • Make sure your NPC or interactive object is set up as a Speaker in Blender so its name and sidekicks are available to the conversation system after import.
  • Keep conversation resources in your Godot project (for example, in res://Conversations/) so they are included when the project is exported.

Start and advance a conversation

  • Trigger a conversation with the Start Conversation command (from a Trigger, UI Option, or scripted call). Provide the list of Conversation files that apply to the Speaker.
  • The Conversation Manager matches the conversation to the characters involved, begins at the initial comment, and emits a conversation_started signal you can hook for UI.
  • When the player chooses a reply, call Select Reply on the Conversation Manager (or wire the input through your dialogue UI) to move to the next comment until the conversation ends.
