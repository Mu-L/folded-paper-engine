Conversations
Speaker setup, dialogue flow, and replies

Set up speakers in Blender

  1. Select the character or object that should speak and enable Speaker in the Folded Paper Engine panel.
  2. Give each speaker a friendly name; that name shows up in the dialogue UI so the player knows who is talking.
  3. Export to GLB with Custom Properties enabled so the speaker tags reach Godot.

Create conversation data

  • Conversations are made up of comments (lines) and replies (player choices). Each reply points to the next comment so you can branch or end a thread.
  • Make sure the conversation references the correct speakers; participants are matched to the speaker names you set in Blender.
  • For NPC-initiated chats, start the conversation from a trigger or interaction command so it opens when the player is close.

Run and advance dialogues

  • Use a trigger or an Event Command to start a conversation by ID. The engine builds a conversation instance and tracks the current comment.
  • When the player picks a reply, advance using the reply’s target comment. When no replies remain, the conversation ends automatically.
  • Pair conversations with a UI prompt (for example, “Press Use to talk”) so players know when a speaker is interactable.
