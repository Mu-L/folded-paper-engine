Conversations
Friendly dialogue setup for creators

Prepare your characters

  • Give each speaking character a clear object name in Blender so the conversation shows the right speaker labels in Godot.
  • Use the Character Controls panel on NPCs and the Player Controls panel on the player so names and sidekicks are registered by the engine.
  • Add a Speaker checkbox to a character if you also want positional voice or sound to play from that actor during a conversation.

Create conversation resources in Godot

  1. In Godot, create a new Conversation resource (Conversation.tres) and fill in comments and replies. Use the required/optional characters fields to tie lines to specific speakers.
  2. Save conversation files somewhere in your project (for example under res://Conversations/).
  3. Optionally add event data to a comment if you want it to trigger an action (like opening a door) when that line appears.

Start a conversation from the game world

  1. In Blender, pick the trigger, character, or object that should start talking.
  2. Open the FPE command list and enable Start Conversation. Add one or more .tres/.res conversation files. The engine chooses the best match based on the character names involved.
  3. Export and run the scene. When the trigger fires (player entry, interact key, or another event command), the conversation manager loads the resource, shows the first comment, and waits for your reply selection.

Advancing replies

  • Use your UI prompt or input handler to call Select Reply on the conversation manager. The built-in manager keeps track of the current comment and stops automatically when there are no replies.
  • Attach your UI (choices, next button) to the conversation manager signals if you want to animate portraits, subtitles, or sound on each line.

Troubleshooting conversations

  • If a conversation does not start, confirm the trigger actually fires and that the Start Conversation command points to existing files.
  • If the wrong speaker names show up, rename the Blender objects for the owner/initializer or adjust the required character names inside the conversation resource.
  • If events tied to comments are not running, check that the comment’s event data is filled in and that the target nodes exist in the scene.
