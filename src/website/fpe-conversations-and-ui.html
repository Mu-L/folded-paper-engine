Conversations and UI
Dialogue data, cursors, and UI options

Conversation system

  • Conversation data includes comments, replies, required characters, and optional sidekicks. Instances receive participant names from the conversation owner and initializer.
  • When started (via command or script), a conversation instance is created with a unique ID and tracks the current comment until no replies remain.
  • Use ConversationUtils.select_reply to advance based on player input and mark completion when replies end.

UI cursors and options

  • UI options registered by the engine are stored in globals so you can render menus or toggles based on the available entries.
  • Cursor objects can be tracked globally; the engine captures the mouse when active and restores the prior mode on exit so editors stay usable.
  • Combine trigger events with UI prompts (e.g., “Press Use”) by monitoring player tracking areas and gating input via the global UI deactivate flag.

Prompts and overlays

  • Mark UI elements in Blender to keep them screen-aligned in Godot. HUD planes or text can follow the camera for diegetic overlays.
  • Use groups and ScriptPath to attach custom UI scripts to imported nodes if the built-in options do not cover your layout.
